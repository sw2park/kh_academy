c. 선언형 자바스크립트 코드

2. JSX란?
b. React에서 사용가능한 비표준형 특정 문법

3. React는 왜 컴포넌트가 전부인가
a. 최종적으로 모든 UI는 다중 컴포넌트들로 구성되고, UI를 결국 컴포넌트의 조합으로 만들게 되기 때문에

4. 선언형의 의미는?
c. 목표로 하는 결과물(UI)를 정의하고 React라이브러리가 단계를 밟을 수 있도록 해주는 것

5. React 컴포넌트란?
a. 출력되어야 하는 HTML코드(JSX)를 반환하는 자바스크립트 함수

6. 하나의 React 애플리케이션은 몇개의 커스텀 React 컴포넌트들을 가질 수 있는가
c. 원하는 만큼

7. 다음중 옳은 것은?
b. React를 이용하여 하나의 DOM노드에 마운트 되는 하나의 루트 컴포넌트를 포함하는 컴포넌트 트리를 만들 수 있다

8. 컴포넌트 트리란 무슨 의미인가?
a. 아래에 더 많은 컴포넌트를 가지고 있는 하나의 root 노드를 의미한다

9. 컴포넌트끼리는 데이터를 어떻게 전달하는지 기술하시오.
프랍스(props, properties)라는 것을 이용해서 객체형 데이터들을 컴포넌트의 파라미터 성격으로 전달된다.
JSX코드에서 마치 HTML코드의 속성값(attribute)을 이용하듯이 데이터를(특히 객체) 기입을 해주면 이 기입을 당한 컴포넌트는 내부에서 해당 데이터를 접근할 수 있다.
이 전달되는 속성을 프랍스라고 부르고 이 프랍스를 이용해서 전달받은 컴포넌트는 해당 데이터를 'key'라는 형태로 사용하면 된다.

10. React 컴포넌트에서 동적인 데이터를 어떻게 출력하는가?(반환되는 JSX 코드 안에서)
a. JSX코드 안에서 중괄호와 JS 표현식을 이용해서 사용할 수 있다. (자바스크립트 표현식)

문제 원문
1. React.js는 어떠한 형식의 코딩 방식을 사용하는가?

b. 명령형 자바스크립트코드
c. 선언형 자바스크립트 코드

2. JSX란?
a. 표준 자바스크립트 문법
b. React에서 사용가능한 비표준형 특정 문법
c. React 함수로 전달할 수 있는 특정한 문장

3. React는 왜 컴포넌트가 전부인가
a. 최종적으로 모든 UI는 다중 컴포넌트들로 구성되고, UI를 결국 컴포넌트의 조합으로 만들게 되기 때문에
b. React 프로젝트들은 컴포넌트를 통해서만 동작하도록 되어 있기 때문에 React코드를 작성할 때 꼭 사용하여야 한다
c. 컴포넌트를 사용하지 않는 '표준 UI'보다 성능상의 이점이 있기 때문에

a. 명령형과 같은 의미
b. 목표로하는 결과물(UI)를 위해 필요한 각각의 단계를 정의하는 것
c. 목표로 하는 결과물(UI)를 정의하고 React라이브러리가 단계를 밟을 수 있도록 해주는 것

5. React 컴포넌트란?
a. 출력되어야 하는 HTML코드(JSX)를 반환하는 자바스크립트 함수
