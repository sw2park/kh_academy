1. Property 
- JS 함수에서 파라미터를 프로퍼티 또는 프랍으로 부른다.
- 파라미터, 매개변수와 같은 동일한 개념을 갖고있다.
- 여러가지를 담을 수 있는데 객체 형태를 갖고 있음
	props = { title : '컴포넌트',
		 description : '어쩌구저쩌구',
		 image : {componentsImg}
		}
- 매개변수의 이름은 아무거나여도 상관없음.

2. 전개 연산자 = 스프레드 연산자
- ...배열이름[인덱스] << ES6 최신버전에 있는 연산자
- 배열이던 객체던 내부의 요소들을 전부 풀어넣는다는 느낌이다.
- 모든 배열이나 객체 요소들을 전달하고 싶을 때 사용가능
	const numbers = [1,2,3];
	ex) const copiedNumbers = [...numbers] // copiednumbers[0] = 1 ... 

3. ES6 - 아래는 블로그 : https://velog.io/@kim_unknown_/JavaScript-ES6
- ES6(ECMAScript 6)는 2015년에 도입된 자바스크립트의 6번째 표준안이다. 
- 현대적인 코드를 사용하면, 코드가 간결해지고 생산성이 향상된다. 
- 이것이 ES6를 사용해야 하는 이유이다.

4. 구조분해 - destructure
- 구조가 영어로 스트럭처인데, 분해를 하겠다해서 de가 붙는다.
- 그래서 destructure, destructing 등 여러 용어로 사용하고 있다.
- 배열이나 객체의 요소를 추출을 쉽게하기 위해 사용하고 있다.

5. 리터럴
- 1옆에 ~에 있는 `요거를 백틱이라고 한다.
- 이걸 이용해서 `$이름 = {변수명}` 문자열 안에 데이터 출력하는것을 리터럴이라고 한다.

6. 그 외 기본적인 내용들
- 자바스크립트에서는 null과 undefined가 있는데, null 그자체를 데이터로 생각하고 있고, undefined는 다른 언어의 null과 같은 의미
- css 분리할 때, 분리한 css 파일도 전체 문서 css에 영향을 준다.
